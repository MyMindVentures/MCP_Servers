# API coverage report

This folder can contain:

- **API_COVERAGE_REPORT.md** — Generated by `node scripts/check-api-coverage.js`. Lists each MCP server’s tool count and, when `docs/expected-endpoints.txt` exists per server, which expected endpoints have no matching tool.
- **expected-endpoints.txt** (inside each server’s `docs/`) — Optional. One endpoint identifier per line (e.g. `list_vaults`, `get_item`). Lines starting with `#` are ignored. Used by the coverage script to flag missing tools.

## Running the coverage check

From the **MCP Servers** root:

```bash
node scripts/check-api-coverage.js
```

The script scans all `MCP_Server_*` directories (except Template and Template_Lightweight), reads `tools/paths.js` to get the list of tools, and optionally compares against `docs/expected-endpoints.txt` inside each server. Results are printed to stdout and written to `docs/API_COVERAGE_REPORT.md`.

## Adding expected endpoints

1. In a server’s `docs/` folder, create `expected-endpoints.txt`.
2. Add one endpoint identifier per line, matching the naming used in the official API or your API_REFERENCE.md (e.g. `list_vaults`, `create_item`). Use `#` for comments.
3. Re-run `node scripts/check-api-coverage.js` to see if any expected endpoints are missing a tool.

You can derive the list from the official API reference, OpenAPI spec, or the “Not covered” rows in that server’s `docs/API_REFERENCE.md`.

## CI

In CI, run:

```bash
node scripts/check-api-coverage.js
```

and optionally fail the build or report status when `API_COVERAGE_REPORT.md` lists missing expected endpoints for a server that has `docs/expected-endpoints.txt`.
